# 02 · UX

## Routes (v1)

| Route | Render | Auth | Purpose |
|---|---|---|---|
| `/` | SSG → ISR (revalidate 5 min) | public | Today's course hero + 6 recent cards |
| `/library` | SSR | public | All courses, search + topic filter + date filter |
| `/course/[slug]` | SSG fallback | public | Lesson reader + quiz |
| `/about` | SSG | public | What CAD is, FAQ, RSS pointer |
| `/api/courses` | edge | public | GET — paged JSON for `/library` |
| `/api/courses/[slug]` | edge | public | GET — single course (for client-side rehydration) |
| `/api/cron/daily` | nodejs | `CRON_SECRET` bearer | POST — daily generation trigger |
| `/login` | SSR | public | Supabase magic-link |
| `/auth/callback` | edge | n/a | Supabase OAuth handler |
| `/admin` | SSR | allowlist (`mat@matsiems.com`) | Preview tomorrow's draft, force-regen, edit metadata |
| `/admin/runs` | SSR | allowlist | Daily cron run log + cost ledger |

`/me` (progress dashboard) is **v2**, not v1.

## Components (src/components/)

| Component | Used on | Responsibility |
|---|---|---|
| `DailyHero` | `/` | Today's course — cover image + title + tagline + CTA to read |
| `CourseCard` | `/`, `/library` | Grid card — cover thumb + title + topic chip + duration |
| `LessonReader` | `/course/[slug]` | Renders one lesson at a time, prev/next nav, scroll progress, "mark complete" |
| `QuizBlock` | `/course/[slug]` (after last lesson) | 5-question multiple choice, score on submit, explanations on review |
| `LibraryFilters` | `/library` | Text search (client-side debounce), topic chips, date-range |
| `TopicChip` | many | Coloured pill — business / tech / language / soft-skill / … |
| `Header` | layout | Logo + nav + (login/avatar) |
| `Footer` | layout | RSS + GitHub + GA4 consent re-open |
| `ConsentBanner` | layout | GA4 Consent Mode v2 — gates `gtag` until accepted |

## Journey — anonymous reader (golden path)

```
1. Lands on /  (organic search "history of negotiation" — today's course)
   └─ DailyHero shows cover + title + "Read in 15 min" CTA
2. Clicks → /course/today's-slug
   ├─ LessonReader opens at lesson 1
   ├─ Scrolls through lesson body
   ├─ Hits "Next lesson" → lesson 2
   └─ … repeats for 10–15 lessons
3. At end → QuizBlock reveals
   ├─ Picks answers, submits → score 4/5
   └─ Explanations show beneath each question
4. Footer CTA → "Tomorrow's course → /library"
5. /library — sees yesterday's, last week's, can search by topic
```

## Journey — admin

```
1. /login → magic-link to mat@matsiems.com
2. → /admin
   ├─ Tab: "Today's course" — preview + force-regen button (calls
   │       /api/cron/daily?force=true with secret)
   ├─ Tab: "Tomorrow" — draft topic list (next-3 candidates)
   └─ Tab: "Runs" — cost ledger, last 30 cron runs, errors
3. Force-regen if Mat doesn't like today's draft → new course written to
   Mongo (same _id, overwrites)
```

## Mobile (≤ 640px)

- DailyHero stacks vertically (cover on top, copy below)
- Library grid is 2 columns
- LessonReader takes the full viewport width, nav buttons stick to
  bottom edge in a 56px-tall bar
- Filters in `/library` collapse to a single "filter" sheet (slide-up)

## Accessibility

- All buttons / links have visible focus rings (Tailwind `focus-visible`)
- Lesson body uses `prose` typography with `max-w-prose` (~65ch)
- Images carry `alt` text — for FLUX covers, the alt is the course
  tagline
- Quiz options are radio inputs with `role="radiogroup"` + labels
- Colour contrast — body text `text-ink` on white = 16.1:1; brand
  `#7c3aed` on white = 4.94:1 (AA for non-large text)
- Skip-to-content link at top of `<body>` for keyboard users
- `prefers-reduced-motion` disables the intro loop autoplay

## Visual tone

- Accent **violet `#7c3aed`** — distinct from the pink "day" cluster
  (yb100/fs/sp/xmas) and from the green "spatial" cluster
  (wbp/ybl/fi/mtd/lituk/wsl). Violet signals education/learning.
- Cover images are FLUX-rendered, 1024×640, vibrant editorial style
  (similar to PAD-Research blog covers)
- Typography: system sans for UI, serif (Georgia stack) for lesson body
- No carousel / no autoplay video on `/` — intro loop is opt-in click
